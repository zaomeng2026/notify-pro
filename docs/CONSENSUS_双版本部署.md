# CONSENSUS_双版本部署

## 明确需求描述

- 形成两条可独立交付线：
  - `LAN 版`：门店本地一体化部署，强调易装、稳定、低门槛。
  - `CLOUD 版`：云端多商户管理，强调账号体系、数据隔离、远程运维。
- 两个版本共享前后端主要代码，但通过“部署模式 + 数据隔离策略”分支行为。

## 技术实现共识

### 1) 统一代码基线

- 服务端维持 `Node.js + Express`，避免双仓维护成本。
- 引入部署模式变量：
  - `DEPLOY_MODE=lan|cloud`
- 模式差异通过配置驱动，不做两套独立代码仓库。

### 2) 数据层

- LAN 版：
  - 默认本地 JSON（可选 MySQL）。
  - 单店铺数据，不做 tenant 分区。
- CLOUD 版：
  - 必须 MySQL。
  - 业务表增加 `tenant_id` 字段，商户隔离查询。

### 3) 部署与运维（宝塔）

- 宝塔面板统一方式：
  - Node 项目管理器/PM2 守护
  - Nginx 反代
  - MySQL 统一初始化脚本
- 必须提供：
  - 环境变量模板（LAN/CLOUD）
  - 标准启动命令
  - 升级与回滚步骤

## 技术约束

- 不破坏现有 API 基础能力（通知上报、配对、大屏展示）。
- 保持前端现有可视化配置可用。
- 所有新增文档和配置文件使用 UTF-8（无 BOM）。

## 边界限制

- 本阶段先完成“可部署架构 + 数据模型与接口骨架”。
- SaaS 高级功能（计费、代理体系）放入后续里程碑。

## 验收标准

- `DEPLOY_MODE` 生效并可区分运行行为。
- 宝塔部署文档可直接执行。
- 云版数据模型具备多租户扩展位。
- 现有单店铺流程不回归（通知上报、统计展示、配对）。
