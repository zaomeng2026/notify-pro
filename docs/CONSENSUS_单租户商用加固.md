# CONSENSUS_单租户商用加固

## 共识结论

- 先做“商用稳定性增强”，不改业务主流程。
- 增强点以低风险增量方式接入，确保不影响当前商家使用。

## 技术方案

1. 原子写入
- `writeJson` 改为“先写临时文件再替换”。

2. 自动备份
- 备份 `settings/pairing/pairing_sessions/records`。
- 提供后台接口手动触发备份。
- 自动清理旧备份（保留最近 N 份）。

3. 诊断接口
- 新增管理员诊断接口，输出：
  - 基础地址有效性
  - 后台密码状态
  - 绑定设备状态
  - 存储后端状态
  - 可执行建议

4. CSV 导出
- 新增管理员导出接口，支持下载收款记录 CSV。

## 技术约束

- 兼容现有前端和手机端，不破坏现有 API。
- 所有新增能力默认可直接在宝塔部署使用。
