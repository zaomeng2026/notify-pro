# 修复清单与验收（2026-02-23）

## 一、为什么之前反复修不好

- 不是单一问题，而是三类问题叠加。
- 编码/内容损坏：前端页面和 `data/settings.json` 出现乱码或非法 JSON。
- 启动链路不统一：`localhost`、`0.0.0.0`、局域网 IP 混用，造成看起来不一致。
- 可见性不足：连接状态不直观，手机/电脑都不知道是否真正连上。

## 二、已修复项

- [x] `server.js`：读取 JSON 失败时自动备份损坏文件并回写 fallback，避免长期回退默认值。
- [x] `data/settings.json`：修复为合法 JSON。
- [x] `run-server.bat`：未传入 `PUBLIC_BASE_URL` 时自动识别局域网 IP，不再默认只用 localhost。
- [x] `一键启动.bat`：显式把 `PUBLIC_BASE_URL` 传给新窗口服务进程，默认打开首页 `/`。
- [x] `一键启动_修复.bat`：补齐可执行内容，作为独立启动脚本可直接用。
- [x] `public/admin.html`：增加连接状态红绿灯；修正设备列表异常字符显示。
- [x] `README.md`：改为中文说明，统一安装和排障口径。

## 三、现场验收步骤（商家安装前必跑）

1. 双击 `一键启动.bat`。
2. 观察控制台输出：
- `Base URL` 必须是局域网 IP（如 `http://192.168.x.x:3180`）。
- 打开的默认页面应为首页 `/`（不是 `/admin`）。
3. 打开后台 `/admin`：
- 保存店铺信息后刷新，数据不能丢。
- 生成绑定二维码，能看到绑定链接和会话状态。
4. 手机扫码授权后运行 Auto.js 脚本：
- 日志出现 `apiUrl=http://同一个局域网IP:3180/api/notify`。
- 后台连接状态显示在线（绿灯）。
- 大屏左上角连接灯为绿灯。
5. 在手机触发一条真实到账通知：
- 大屏新增记录且时间、金额、渠道正确。

## 四、若再次出现“保存后还原默认”

- 先检查 `data/settings.json` 是否为合法 JSON。
- 若文件被自动备份，目录内会出现 `settings.json.broken.<时间戳>.txt`。
- 用后台重新保存一次配置，确认文件恢复。
