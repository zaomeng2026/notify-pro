# 6A 商业版实施方案

## 1. Align

- 目标: 从“可用工程样机”升级为“可交付商家产品”。
- 用户画像: 非技术商家，要求低学习成本、低维护。
- 验收标准:
  - 3 分钟完成首次绑定
  - 商家无需手输 URL
  - 开机后自动恢复运行

## 2. Architect

- 架构:
  - 手机 Auto.js -> 局域网 API -> 本地服务 -> 大屏浏览器
- 关键模块:
  - 配对服务: 旧版 code claim + 新版 QR auto claim
  - 展示服务: SSE 实时刷新
  - 配置服务: 商家后台

## 3. Atomize

- 已落地:
  - 扫码会话创建 API
  - 扫码授权 API
  - 自动认领 API
  - 管理后台二维码生成与状态轮询
  - 手机扫码确认页
  - 安装向导页
  - Auto.js 自动认领脚本模板
- 待落地:
  - 打包为单文件 exe
  - 开机自启安装器
  - 云端远程运维（可选）

## 4. Approve

- 风险与对策:
  - 网络隔离导致配对失败: 强制同网段 + 健康检查
  - 扫码后未自动认领: 后台显示会话状态并提示脚本重启
  - 安全风险: 推荐启用 auth token

## 5. Automate

- 标准安装 SOP:
  1. 启动服务
  2. 打开 /admin 填写店铺信息
  3. 生成二维码并扫码授权
  4. 启动 Auto.js 自动版脚本
  5. 打开大屏 / 并全屏展示

## 6. Assess

- 当前达成度:
  - 功能达成: 80%
  - 商业交付感: 70%
- 下一阶段优先级:
  1. 免 Node 交付（exe）
  2. 开机自动拉起服务+全屏
  3. 一键诊断工具（端口、防火墙、网络）
