# ALIGNMENT_双版本部署

## 原始需求

- 使用 6A 工作流设计并推进改造。
- 同时支持两个版本：
  - 单机局域网版（门店本地运行，低成本、离线可用）
  - 云服务器版（多商户 SaaS）
- 部署管理需对宝塔面板友好，安装维护尽量简化。

## 项目现状理解

- 现有服务端为 `Node.js + Express`（`server.js`），已具备：
  - 收款通知接入 `/api/notify`
  - 配对二维码流程（含首页未绑定引导）
  - 管理端配置与大屏展示
  - 记录存储 JSON / MySQL 回退能力
- 现有架构偏“单租户单店铺”，不具备完整多租户账号体系。
- 已有 Android 客户端工程（`app/`）和 Auto.js 脚本（`autojs/`）。

## 任务边界

### 本轮必须交付

- 双版本技术方案文档（局域网版 + 云服务器版）。
- 宝塔部署标准化方案（目录结构、启动方式、配置文件、运维流程）。
- 代码层面先完成“可并行支持双版本部署”的基础改造骨架，不一次性重构全部 SaaS 功能。

### 本轮不一次性完成

- 完整 SaaS 商户中心（计费、工单、权限细粒度 RBAC、审计全链路）。
- 大规模分布式能力（多地域、容灾双活、消息队列削峰）。

## 关键风险

- 直接“单租户 -> 多租户”全量改造风险高，容易破坏现有稳定功能。
- 宝塔环境变量与 Node 进程管理如果不标准化，后续商户安装会继续混乱。
- 云端版本若无租户隔离设计，数据安全不可接受。

## 关键决策点（需你确认）

1. 云版本是否必须“多商户完全隔离（tenant_id）”？  
   - 建议：必须，数据库所有业务表带 `tenant_id`。
2. 商户账号体系是否采用“手机号+密码（或验证码）登录”？  
   - 建议：先“账号密码 + JWT 会话”，验证码后续迭代。
3. 云端是否要求“一个商户可绑定多台手机设备”？  
   - 建议：支持，多设备并发上报。
4. 局域网版是否继续允许“无密码后台”？  
   - 建议：默认启用后台密码，首次启动强制设置。
5. 宝塔部署目标系统优先级：`Ubuntu 22.04` 还是 `CentOS`？  
   - 建议：优先 Ubuntu 22.04（生态更稳定）。

## 验收标准（本轮）

- 文档：`CONSENSUS/DESIGN/TASK` 可直接指导开发。
- 代码：项目可明确区分 `LAN` 与 `CLOUD` 部署模式。
- 运维：提供宝塔可执行的部署步骤，含 MySQL 初始化、Node 启动、反向代理、升级流程。
