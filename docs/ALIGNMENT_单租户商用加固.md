# ALIGNMENT_单租户商用加固

## 原始需求

- 选择路线 A：先把单租户版本做到可商用稳定，再考虑 SaaS。
- 目标是降低故障率、降低商家运维门槛、提高数据可靠性。

## 项目现状

- 已具备收款通知上报、配对绑定、大屏展示、后台配置。
- 支持 JSON/MySQL 记录存储，部署已区分 `lan/cloud` 模式。
- 当前仍缺少“商用运维常用能力”：诊断、导出、备份防护。

## 本轮边界

### 本轮做

- 数据文件写入可靠性加固（避免 JSON 写坏导致不可用）。
- 自动备份与保留策略（面向单机门店部署）。
- 诊断接口（快速判断为何“连不上/显示离线/配置异常”）。
- 记录 CSV 导出（便于商家留档核对）。

### 本轮不做

- 多租户登录与租户隔离（后续 SaaS 阶段）。
- 支付通道 SDK 深度接入（仍基于通知监听链路）。

## 验收标准

- JSON 写入失败率显著降低（原子写流程）。
- 可通过接口触发与查看备份结果。
- 管理员可导出收款 CSV。
- 诊断接口可返回明确的 warning/error 列表。
