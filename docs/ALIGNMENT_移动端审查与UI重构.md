# ALIGNMENT_移动端审查与UI重构

## 原始需求

- 使用 6A 工作流审查 bug、功能一致性。
- 手机 App 使用 `ui-ux-pro-max` 思路重新设计界面。

## 项目现状理解

- Android 客户端为 `ViewBinding + XML`（非 Compose）。
- 现有页面功能较全，但信息密度过高，交互分组弱，可读性和操作路径不够清晰。
- 服务端已启用后台密码鉴权，`/api/settings` 写入需管理员密码头。

## 已识别关键问题

1. 一致性缺陷：App 保存设置未携带管理员密码头，后台设置密码后手机端保存会失败。
2. UI/UX 缺陷：单页堆叠过长、按钮层级混杂、状态信息与操作未分区。
3. 功能跟进缺口：服务端新增备份参数（保留份数/小时）后，App 未提供对应编辑入口。

## 任务边界

- 本次仅改 Android App（`app/`）：
  - 修复一致性 bug
  - 重构主界面布局
  - 增加备份参数配置输入项
- 不改 Auto.js 脚本协议，不改服务端业务流程（仅适配已有接口）。

## 验收标准

- 设置管理员密码后，App 仍可成功保存店铺设置。
- 界面分区清晰，核心操作（扫码绑定、权限、连接测试）在首屏可见。
- 备份参数可在 App 读取与保存。
- Android 工程可通过 Kotlin 编译检查。
